# 🌍 Серверная часть системы «Эко-климат»

Система автоматического мониторинга параметров микроклимата (температура, влажность, CO2) в учебных и офисных помещениях.
Проект решает проблему «синдрома закрытого помещения», помогая контролировать качество воздуха для поддержания продуктивности.

## 🛠 Технологический стек

Проект построен на базе современной инфраструктуры для работы с временными рядами и IoT:

*   **Eclipse Mosquitto** — MQTT-брокер для получения данных с датчиков.
*   **FastAPI** — основной бэкенд для обработки полученных данных.
*   **TimescaleDB (PostgreSQL)** — база данных, оптимизированная под хранение метрик.
*   **Apache Superset** — платформа для визуализации данных и создания дашбордов.
*   **Docker & Compose** — контейнеризация всей инфраструктуры.

## 🚀 Быстрый старт

### Требования
*   Установленный [Docker Compose](https://docs.docker.com).

### Установка и запуск

1. **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/linar-mukhamedyarov/eco-climate.git
    cd eco-climate
    ```
2. **Настройте переменные окружения:**
    Отредактируйте файл `.env`. Обязательно укажите путь для хранения данных БД, к которому у Docker будет доступ на запись:

    ```bash
    DB_LOCATION=/home/user/eco-data/db  # Укажите свой путь
    ```
3. **Соберите и запустите контейнеры:**
    ```bash 
    docker compose build
    docker compose up -d
    ```

## 📊 Документация и визуализация
*   **Apache Superset:** http://localhost:8088 — визуализация графиков и аналитика (логин/пароль по умолчанию настраиваются в `.env`).
*   **MQTT Брокер:** доступен на стандартном порту 1883.

## 🏗️ Архитектура системы
1. Модули на esp32 отправляют данные по протоколу MQTT в брокер Mosquitto.
2. FastAPI сервис (Consumer) подписывается на топики, обрабатывает данные и сохраняет их в TimescaleDB.
3. Apache Superset подключается к БД и строит графики в реальном времени.

## 📡 Протокол обмена данными (MQTT)
FastAPI автоматически подписывается на топик по шаблону `/room/#`.

### Формат топиков:
Данные должны отправляться в топик следующего вида:
`room/{room_id}`

**room_id** — уникальный идентификатор помещения (например, `101`, `lab_5`).

**Пример:** `room/402` — передача данных климата 402 кабинета.

### Формат сообщения (Payload):
Система ожидает данные в формате **JSON**:
```json
{
  "temperature": 22.5,
  "humidity": 50.5,
  "co2": 500
}
```
